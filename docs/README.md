## 📋 문서 작성 규칙

### Constitution v1.2.1 준수
본 문서 구조는 [42 miniRT Constitution v1.2.1](../.specify/memory/constitution.md)의 Principle VII을 따릅니다.

### 파일 명명 규칙
- 모든 파일명은 한글과 언더스코어(_) 사용: `파일_이름.md`
- 구현 요약은 이슈 번호 접두사 사용: `NNN_기능_설명.md`
- 날짜 기반 파일은 YYYY-MM-DD 접두사 사용 (필요시)

### 문서 위치 결정 가이드

#### 새 기술 문서 작성 시
- **알고리즘/이론** → `00-전체설명/`
- **사용법/튜토리얼** → `01-시작하기/`
- **개발 도구/프로세스** → `02-개발/`
- **특정 기능 상세** → `03-기능/[하위시스템]/`
- **릴리스 관련** → `04-릴리스/`

#### 프로젝트 진행 중
- **현재 상태 업데이트** → `05-프로젝트_히스토리/` (기존 파일 수정)
- **이슈 완료 요약** → `05-프로젝트_히스토리/구현_요약/NNN_제목.md`

#### 문서 생명주기
1. **작성** → 적절한 카테고리에 배치
2. **유지보수** → 지속적 업데이트
3. **완료/오래됨** → `archive/YYYY-MM-legacy/`로 이동

### 상호 참조
- 문서 간 링크는 상대 경로 사용
- 예: `[파일 구조](../02-개발/파일_구조.md)`
- 외부 링크는 절대 URL 사용
